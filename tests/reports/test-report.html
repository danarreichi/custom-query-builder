Full driver parity: MySQL (mysqli) vs SQLite (sqlite3)

CompiledSqlTest, EdgeCaseTest, and ExecutionTest each have a SQLite twin sharing the same scenarios via PHP traits (ExecutionScenariosTrait, EdgeCaseScenariosTrait) or a mechanically-generated double-quote variant (SqliteCompiledSqlTest). Now includes the new paginate() feature (page-based pagination built on calc_rows()'s portable COUNT(*) mechanism) and the calc_rows()/get_with_calc_rows() simplification that dropped SQL_CALC_FOUND_ROWS entirely. 126 tests run on each driver — full parity, not a partial smoke suite. Expected/Result columns are extracted directly from each test's own assertion calls (statically parsed from the real source, not hand-written) — Expected shows what the assertion checks for, Result confirms the real PHPUnit outcome for that assertion.

Why two separate PHPUnit runs, not one combined command: CodeIgniter 3's own DB_driver.php::escape_identifiers() caches its quote-char regex in a function-local static variable — shared process-wide across every driver instance, not per-connection. Running a mysqli-backed test class and a sqlite3-backed one back to back in the same PHP process lets whichever connects first "win" that cache, silently corrupting the other driver's exact-compiled-SQL assertions. tests/phpunit.xml now defines two named testsuites (mysql, sqlite) specifically so they're never invoked together — this is a CI3 framework quirk, not a bug in this library.
